Demonstrated Capabilities

Key areas of engineering excellence integrated inside this platform

🔑 Authentication

Strict Google OAuth 2.0 federated login interface. Fully removes insecure local forms and password storage, adhering to zero-trust passwordless authentication.

🛡️ Security

Layered defenses including Double-Submit WTF-CSRF protection, rate limiting on sensitive forms, automatic 15-minute inactivity timers, and Secure/HttpOnly session cookies.

💳 Payments

PayPal checkout integration combined with static UPI QR payments, demonstrating secure external payment flows.

🚢 Deployment

Dockerized container stack utilizing Gunicorn WSGI servers and a custom REST health check endpoint (/health) to verify environmental diagnostics.

💎 UI/UX Design

Modern glassmorphic Stripe/Linear design style featuring full prefers-reduced-motion support, keyboard outline accessibility, and a seamless Dark/Light theme toggler.

📁 Database Management

Relational SQLite schema featuring automated column migrations, telemetry logging counters, and data privacy cleanup jobs executing on request triggers.

Project Case Study

🔍 Why This Project Was Built

Designed to serve as a comprehensive staff-level engineering artifact. It bridges the gap between typical tutorial apps and complex enterprise SaaS systems by emphasizing defensive security, clean modular routes, and a premium visual aesthetic.

🛠️ Technologies & Packages

Built with Flask as the microframework, SQLite for database isolation, and security extensions (WTForms for CSRF protection, Limiter for rate limits). Local QR rendering is driven by qrcode and pillow.

🛡️ Security Challenges Solved

Mitigates vulnerabilities including SQL Injection (parameterized queries), CSRF (WTF-CSRF middleware), session hijacking (HttpOnly & SameSite cookies), and brute-force vectors (rate-limiting endpoints).

🚢 Deployment Readiness

Fully containerized with a Dockerfile and docker-compose.yml configuration using Gunicorn for production execution. Includes a secure `/health` checks router suited for cloud environments like Render and Railway.

🎓 Educational Purpose

Serves as a safe demonstration platform for verifying real-world payment SDK workflows, secure session logic, and OAuth callback handling in Python, providing recruiter-ready technical transparency.

⚙️ Automated Administration

Equipped with a secure role-based access control (RBAC) mechanism. Restricts administrator privileges dynamically to the designated owner email read from environment parameters.

Session Expiration Inactivity logout cleans session data after 15 minutes of idle status. Restricts window for session theft on shared client systems. ✓ Active
Environment Variables Loads payment client tokens, keys, and DB paths via python-dotenv, excluding real keys from source. Zero hardcoded secrets. Avoids token leaks. ✓ Active
